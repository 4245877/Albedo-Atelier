# Безопасность фермы: модель развёртывания и чеклист

Ферма управляет **физическими устройствами с нагревателями**. «Доверенный LAN»
— это предположение, а не гарантия: любой, кто достигает control plane, может
запускать/останавливать печать. Ниже — что обеспечивает сам софт и что обязан
сделать оператор снаружи.

## Что обеспечивает софт (fail-closed по умолчанию)

| Механизм | Поведение |
| --- | --- |
| Auth | Без `ORCHESTRATOR_API_TOKEN` все мутации отклоняются 503. Обход — только явный `ALLOW_UNAUTHENTICATED_MUTATIONS=1`. |
| CSRF | `Origin`-проверка + `Sec-Fetch-Site: cross-site` → 403 на мутации. |
| DNS rebinding | Host-allowlist: IP-литералы/localhost/имя сервиса проходят, любые другие DNS-имена — только через `ALLOWED_HOSTS`. |
| Clickjacking | `X-Frame-Options`, `CSP frame-ancestors` на API (DENY) и на дашборде (SAMEORIGIN). |
| Прокси-заголовки | `X-Forwarded-*` не участвуют ни в одном доверительном решении (trusted-proxy не сконфигурирован). |
| go2rtc | Наружу — только сигнальный WS с `src`-allowlist (`k2`), `dst` запрещён, HTTP API не проксируется, порт 1984 — loopback. |
| Порты | Оркестратор не публикуется на хост вообще; дашборд по умолчанию `127.0.0.1:8090` (`DASHBOARD_BIND` — явное решение). |
| Bambu TLS | MQTT без проверки сертификата — только явный opt-in (`allowInsecureTls` / `BAMBU_ALLOW_INSECURE_TLS=1`). |
| Unattended dispatch | Только SQLite-задание с hash-артефактом, свежим `schedulable`-анализом, явными `night` + `unattendedAllowed`; иначе отказ. |

## Чего софт НЕ обеспечивает (обязанности оператора)

1. **Moonraker ходит по HTTP без аутентификации.** Любой в том же сегменте
   может управлять принтером напрямую, минуя ферму. Выберите режим:
   - *минимум*: отдельный VLAN «printers» (принтеры + хост фермы), закрытый
     файрволом от остального LAN/Wi-Fi;
   - *лучше*: Moonraker за reverse-proxy с TLS+auth, или доступ только через
     WireGuard/VPN до хоста фермы;
   - *недопустимо*: принтеры в общем/гостевом Wi-Fi.
2. **Общий bearer-token дашборда** — это один секрет на всех операторов, без
   персональных учёток и аудита «кто нажал». Полноценная пользовательская
   аутентификация не реализована — не публикуйте дашборд за пределы
   доверенного сегмента (и тем более в интернет) без внешнего SSO/reverse-proxy.
3. **WebRTC-порт 8555** опубликован на LAN (иначе камера K2 не работает).
   Ограничьте его файрволом до подсети операторов.

## Чеклист файрвол/VLAN/VPN

- [ ] Принтеры (Moonraker/Bambu/Creality) — в отдельном VLAN; из общего LAN к
      ним нет маршрута.
- [ ] К хосту фермы из LAN открыты только: `8090/tcp` (дашборд, если
      `DASHBOARD_BIND=0.0.0.0`) и `8555/tcp+udp` (WebRTC).
- [ ] `3100` (оркестратор) и `1985` (go2rtc API) НЕ слушают на LAN-интерфейсе
      (compose уже так делает — проверьте `ss -tlnp`).
- [ ] `ORCHESTRATOR_API_TOKEN` задан (`openssl rand -hex 24`), продублирован в
      fulfillment.
- [ ] Доступ извне — только через VPN (WireGuard) до хоста; никакого
      проброса портов на роутере.
- [ ] `ALLOWED_HOSTS` перечисляет все DNS-имена, по которым открывают дашборд.
- [ ] `BAMBU_ALLOW_INSECURE_TLS` включён осознанно и только в изолированном
      сегменте.
- [ ] `.env` и `config/printers.json` — права `600`, не в git.

## Незакрытые риски (честно)

- Нет персональной аутентификации и авторизации операторов (один общий токен).
- Moonraker-трафик в VLAN идёт открытым HTTP (спасает только изоляция сегмента).
- Дашборд-страница сама по себе без логина: физический доступ к машине
  оператора = доступ к ферме.
